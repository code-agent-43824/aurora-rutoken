# aurora-rutoken

Тестовое приложение для ОС Аврора, работающее с токенами **Рутокен ЭЦП 3.0** по **USB** и **NFC**.

Стек: нативный Qt/C++ + QML (Аврора SDK, RPM-пакет). Доступ к токену: PKCS#11 (`librtpkcs11ecp.so`) поверх PC/SC (`pcscd`): USB — через CCID-handler, NFC — через NFC-стек ОС (`nfcd`).

## Сборка

RPM собирается автоматически в GitHub Actions при каждом пуше в `main` (workflow [build-rpm.yml](.github/workflows/build-rpm.yml)). Два независимых чистых matrix jobs разворачивают Аврора Platform SDK **5.2.1.200** и отдельно собирают `armv7hl` и `aarch64`; повторное использование object-файлов другой архитектуры исключено. После сборки CI проверяет RPM arch, ELF machine, dynamic loader и метаданные подписи. Для каждой архитектуры CI также скачивает исходный официальный `ru.rutoken.librtpkcs11ecp-2.19.0.0-1` с `download.rutoken.ru`, сверяет закреплённый SHA-256 и проверяет архитектуру/экспорты библиотеки. Публикация происходит только после успеха обеих архитектур. Локально ничего ставить не нужно.

Имя ZIP-артефакта Actions содержит версию, release и архитектуру: например, `rutokentestapp-0.0.3-1-armv7hl-rpms.zip`. Внутри находятся два уже версионированных RPM: официальный PKCS#11-модуль Рутокен и приложение.

Собранный RPM подписывается **общедоступной тестовой парой ключей ОМП** (профиль Regular, `ci/keys/`, источник — [документация по подписанию](https://developer.auroraos.ru/doc/sdk/app_development/packaging/package_signing)) утилитой `rpmsign-external` внутри чрута PSDK — без подписи Аврора 5 отклоняет пакет с ошибкой `BadPackageSignature`. Пакеты на тестовом ключе ОС принимает **только при включённом режиме разработчика**.

Готовые RPM: **[релиз ci-latest](https://github.com/code-agent-43824/aurora-rutoken/releases/tag/ci-latest)** — обновляется каждой успешной сборкой, файлы скачиваются прямо с телефона. Те же пакеты лежат в версионированном артефакте каждого запуска Actions.

## PKCS#11 в v0.0.3

Приложение динамически загружает системный файл `/usr/lib/3rdparty/ru.rutoken.librtpkcs11ecp/librtpkcs11ecp.so`, получает таблицу функций через `C_GetFunctionList`, выполняет `C_Initialize`, читает `C_GetInfo` и корректно вызывает `C_Finalize`. Экран показывает путь, версии Cryptoki и библиотеки, производителя и описание. Если официальный пакет не установлен, приложение всё равно запускается и явно показывает отсутствие библиотеки.

Бинарная библиотека не копируется внутрь MIT-пакета приложения. В релизе рядом публикуется **неизменённый официальный RPM Рутокен**: это сохраняет исходный пакет и соответствует условиям распространения неизменённого ПО из разделов 4.3–4.4 [лицензионного соглашения Рутокен](https://www.rutoken.ru/download/license/License_Agreement_Rutoken.pdf). Официальная страница загрузки: [PKCS#11-библиотека Рутокен](https://www.rutoken.ru/support/download/pkcs/).

## Подтверждено на физическом устройстве

На 32-битном телефоне с Авророй 5.2 версия 0.0.2 показала зелёные статусы `nfcd`, NFC-адаптера, `libpcsclite` и `pcscd`. При подключении Рутокен ЭЦП по USB в списке появляется дополнительный PC/SC-ридер `Aktiv Rutoken ECP 00`. Работа непосредственно через PKCS#11 начнётся в v0.0.3.

## Установка на телефон (Аврора 5.x)

1. **Включите режим разработчика** (Настройки → Приложения/Система → Режим разработчика; потребуется учётная запись Аврора ID или локальное подтверждение — зависит от версии ОС). Без него пакет, подписанный тестовым ключом, не установится (`BadPackageSignature`) или не запустится.
2. Разрешите установку недоверенных приложений (Настройки → Безопасность → Недоверенное ПО).
3. Скачайте из [релиза ci-latest](https://github.com/code-agent-43824/aurora-rutoken/releases/tag/ci-latest) **два** RPM своей архитектуры: для 32-битного устройства — оба `*.armv7hl.rpm`, для 64-битного — оба `*.aarch64.rpm` (пакет чужой архитектуры отклоняется с ошибкой `BadPackageArchitecture`).
4. Сначала откройте в «Файлах» и установите `ru.rutoken.librtpkcs11ecp-2.19.0.0-1.<архитектура>.rpm`, затем `ru.codeagent43824.rutokentestapp-0.0.3-1.<архитектура>.rpm`.
5. Запустите приложение и проверьте четыре новые строки PKCS#11. Альтернатива установке из «Файлов» — `devel-su pkcon install-local <путь-к-rpm>`.
