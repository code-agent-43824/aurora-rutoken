### v0.2 — Вход по PIN и детали токена — В РАБОТЕ с 2026-07-20

Цель: по тапу на карточку токена — экран деталей с расширенной информацией и вводом PIN пользователя; проверка PIN через `C_Login`, индикатор оставшихся попыток.

Архитектурное решение (конфликт контекстов): `C_Initialize`/`C_Finalize` глобальны для процесса; фоновый опрос `TokenWatcher` и операция логина не должны пересекаться. Решение — **общий `QMutex`** вокруг каждого изолированного цикла `init…finalize` (и опрос, и логин берут его). Логин выполняется за один цикл под мьютексом: `C_Initialize → C_OpenSession → C_Login(pin) → [C_GetTokenInfo для флагов] → C_Logout → C_CloseSession → C_Finalize`.
