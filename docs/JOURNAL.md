Свежая инициализация на каждом опросе бесплатно ловит и новый USB-ридер, и NFC-карту. Между потоками разделяется только стабильный указатель `C_GetFunctionList` (только чтение) → без гонок. ## 2026-07-20 — Claude (кодинг-агент), сессия 4: крэш v0.0.3-1 на телефоне, фикс ABI PKCS#11

**Проблема (отчёт владельца):** v0.0.3-1 на телефоне запускается, ~5 секунд показывает спиннер и вылетает; одинаково без токена, с USB- и с NFC-токеном. v0.0.2 работала.

**Диагноз (по коду, воспроизводимо):** `src/pkcs11_minimal.h` объявлял структуры с `#pragma pack(1)`. Однобайтовая упаковка — требование PKCS#11 **только для Windows**; на Linux (и Авроре) библиотеки собираются с естественным выравниванием. Последствия: (1) в `CK_FUNCTION_LIST` настоящий указатель `C_Initialize` лежит по смещению `sizeof(void*)` (4 на armv7hl), а читался со смещения 2 → вызов мусорного адреса → SIGSEGV в рабочем потоке → падение процесса (спиннер до этого момента — поток жив, PC/SC-часть успевает пройти); (2) `CK_INFO` получалась 72 байта вместо 76 (arm32) / 88 (arm64) — `C_GetInfo` писал за границу стекового буфера. Прежние static_assert'ы закрепляли именно ошибочную упаковку, поэтому host-компиляция ничего не ловила.

**Фикс:** pack(1) удалён; static_assert'ы переписаны на естественное выравнивание с ветвлением по `sizeof(CK_ULONG)`/`sizeof(void*)` (armv7hl: flags=36, size=76, C_Initialize=4; aarch64: 40/88/8) — 32-битные значения проверяет компилятор armv7hl прямо в CI. Release пакета поднят до `0.0.3-2`.

**Верификация фикса до пуша:** написан харнесс `abi_test.cpp` (dlopen + наш заголовок), скачана **реальная официальная** `librtpkcs11ecp.so` 2.19.0.0 Linux x86_64 с download.rutoken.ru: `C_GetFunctionList` → interface 2.40, осмысленные указатели; `C_Initialize` → CKR_OK; `C_GetInfo` → «Aktiv Co.», «Rutoken ECP PKCS #11 library», версия 2.19, канарейка рядом с буфером не тронута; `C_Finalize` → CKR_OK. Со старым заголовком эта последовательность падала бы на первом же вызове.

**Урок для будущих агентов:** ABI-заголовки нельзя «проверять» static_assert'ами, выведенными из самого заголовка, — сверяться нужно с реальной библиотекой или официальным `pkcs11.h`; pack(1) в PKCS#11 — только Windows.

## 2026-07-19 — Watson (coding-агент): CI исправлен, v0.0.2 закрыта

**Сделано:** armv7hl и aarch64 перенесены в независимые чистые matrix jobs (`TARGET_ARCH`, один runner — одна архитектура).
