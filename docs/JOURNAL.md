## 2026-07-19 — Claude (кодинг-агент), сессия 2

**Сделано:**

- Проведено исследование по заданию владельца: найдены исходники примеров Авроры на Мос.Хабе (группа https://hub.mos.ru/auroraos; подгруппы demos, examples, flutter, edu), изучены ApplicationTemplate, NfcUseCases (pcsc-lite + nfcd через D-Bus), UsbUseCases, Flutter SDK и плагины сообщества.
- Собрана информация о работе Рутокен на Авроре: стек `приложение → librtpkcs11ecp.so (PKCS#11) → libpcsclite → pcscd → CCID (USB) / NFC-handler (nfcd)`; официальные RPM библиотеки rtPKCS11ECP для Авроры ARM32/ARM64 v2.19.0.0; Рутокен ЭЦП 3.0 NFC официально поддерживает Аврору; Bluetooth-модели не поддерживаются. Всё с источниками — в [docs/RESEARCH.md](RESEARCH.md).
- **Принято решение**: приложение пишем на **нативном Qt/C++ + QML** (не Flutter). Обоснование — [docs/RESEARCH.md](RESEARCH.md) §4 (PKCS#11 — C API, официальный NFC-пример на C++, отсутствие PC/SC-плагинов для Flutter).
- Созданы docs/RESEARCH.md, PLAN.md (план этапов 0–6 с чекбоксами), docs/JOURNAL.md; переписаны CLAUDE.md (правила для агентов: план → действие → фиксация; git без веток) и README.md.
- По требованию владельца установлен порядок работы с git: **без веток, коммиты сразу в `main` с немедленным пушем**. Ветка `claude/init-lr079p` влита в `main` (fast-forward) и удалена локально. Удалить её на GitHub из этой среды не удалось: прокси среды выполнения блокирует удаление веток (HTTP 403 и для `git push --delete`, и для GitHub API). Владельцу нужно один раз удалить её вручную: GitHub → Branches → Delete `claude/init-lr079p`.

**Почему:** первое задание владельца проекта — найти исходники на МосХабе, определить стек (Qt vs Flutter), записать документацию в репозиторий.
